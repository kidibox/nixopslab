# nixopslab

Kubernetes cluster manifests managed with [nixidy](https://nixidy.dev/), using [den](https://den.denful.dev/) for entities/aspects, [flake-file](https://github.com/sini/flake-file) for flake generation, and [sini/files](https://github.com/sini/files) for declarative manifest sync.

## Structure

```
.
├── flake-file.nix          # Source of truth for flake inputs/config
├── flake.nix               # Auto-generated by `nix run .#write-flake`
├── k8s-manifests/          # k8s-manifests factory modules (curried functions)
│   └── argocd.nix           # { cluster, ... }: { config, charts, ... }: { ... }
├── modules/                # Auto-imported via import-tree (flake-parts modules)
│   ├── default.nix           # (generated by flake-file dendritic)
│   ├── argocd/               # den aspect: k8s-manifests class + service-domains quirk
│   └── flake/               # Flake configuration
│       ├── core.nix         # Imports den, files, flake-file (dendritic)
│       ├── devshell.nix     # nixidy CLI + dev shell
│       ├── files.nix        # Walks nixidy envPkg, registers manifests as sini/files
│       ├── inputs.nix       # flake-file declarative input definitions
│       ├── k8s.nix          # k8s.manifests option (attrsOf path, keyed by aspect name)
│       ├── clusters.nix     # den.schema.cluster.isEntity + den.clusters option type
│       ├── nixidy.nix       # Builds nixidy envs from den.clusters + aspect includes
│       ├── policies.nix     # den policies: cluster-aspect auto-include
│       └── entities/
│           └── prod.nix     # Production cluster entity + aspect (den.clusters.prod, den.aspects.prod)
├── apps/                    # Shared nixidy module options (imported by mkEnv)
│   └── default.nix           # networking.domain option
├── clusters/               # Per-cluster nixidy config modules
│   └── prod.nix            # Reads cluster specialArg, sets nixidy.target and domain
└── manifests/               # Generated output (committed for Argo CD)
    └── prod/
```

## Data flow

```
den.clusters.prod (entity)  ──→  extraSpecialArgs.cluster  ──→  config.cluster.* in nixidy modules
      │                                                                    │
      ├── domain, networks, nixidy.target                                  ├── argocd reads config.networking.domain
      └── storage                                                   └── set by clusters/prod.nix

den.aspects.prod (includes) ──→  collectAspectNames  ──→  k8s.manifests.argocd
      │                                                      │
      └── includes = [ den.aspects.argocd ]                  └── import & call with { cluster = ctx }
                                                                    │
                                                              k8s-manifests/argocd.nix
                                                              { cluster, ... }: { config, charts, ... }: { ... }
```

Clusters are den entities (`den.schema.cluster.isEntity = true`). The single source of truth is `den.clusters.<name>` in `modules/flake/entities/`. Each cluster also has a den aspect (`den.aspects.<name>`) with `includes` listing which k8s services to deploy.

k8s-manifests factories live in `k8s-manifests/` (outside `modules/` to avoid import-tree) and are registered in `k8s.manifests.<name>` (a plain flake-parts option). The nixidy env assembler walks the aspect includes chain, collects matching k8s.manifests entries, imports and calls each factory with cluster context.

## Commands

| Command | Description |
|---|---|
| `nix run .#write-files` | Write manifests into the repo |
| `nix run .#diff-files` | Preview what would change (no writes) |
| `nix flake check` | Verify manifests + flake.nix are in sync |
| `nix run .#write-flake` | Regenerate flake.nix from module options |

## Adding a new cluster

1. Create `modules/flake/entities/<name>.nix` — register `den.clusters.<name>` and `den.aspects.<name>` with includes.
2. Create `clusters/<name>.nix` — nixidy module that reads `cluster` specialArg, sets `config.cluster.*`, `config.nixidy.target`, `config.networking.domain`.
3. Run `nix run .#write-files` and commit.

## Adding a new application

1. Create `k8s-manifests/<app>.nix` — curried function `{ cluster, ... }: { config, charts, lib, ... }: { ... }`.
2. Create `modules/<app>/default.nix` — declare `den.aspects.<app>` (with quirks like `service-domains`), `den.classes.k8s-manifests`, and `k8s.manifests.<app> = ./k8s-manifests/<app>.nix`.
3. Add `den.aspects.<app>` to the relevant cluster aspect's `includes` list in `modules/flake/entities/`.
4. Run `nix run .#write-files` and commit.

## Design

- **Den entities**: Clusters are `den.clusters.<name>` with `den.schema.cluster.isEntity = true`. Ready for den's policy engine to route aspects per-cluster.
- **Den aspects**: `den.aspects.<cluster>` with `includes` listing k8s service aspects. The includes chain is walked to collect k8s-manifests factories.
- **k8s.manifests**: Plain `attrsOf path` option (not in den.aspects) because den's `aspectContentType` wraps freeform keys. Factory paths are imported and called with cluster context.
- **Cluster context**: Flows from `den.clusters` → `extraSpecialArgs.cluster` → nixidy module options `config.cluster.*`. Single source of truth.
- **flake-file (dendritic)**: `flake.nix` is auto-generated. Edit `flake-file.nix` or `modules/flake/inputs.nix`, then regenerate.
- **sini/files**: Manifests declared as file entries. `nix run .#write-files` updates them; `nix flake check` verifies sync.
- **import-tree**: modules under `modules/` are auto-discovered. Prefix with `_` to exclude.
- **One nixidy env per cluster**: `clusters/<name>.nix` configures target repo/branch/path.
- **Helm via nixhelm**: All charts pinned in flake lock for reproducibility.
