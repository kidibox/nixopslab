# nixopslab

Kubernetes cluster manifests managed with [nixidy](https://nixidy.dev/), inspired by the [fleet pattern](https://den.denful.dev/) from [sini/nix-config](https://github.com/sini/nix-config).

## Structure

```
.
├── flake-file.nix          # Source of truth for flake inputs/config (edit this)
├── flake.nix               # Auto-generated by `nix run .#write-flake` (do not edit)
├── modules/                # Auto-imported via import-tree (flake-parts modules)
│   └── flake/              # Flake configuration
│       ├── core.nix         # Imports den, files, flake-file (dendritic)
│       ├── devshell.nix     # nixidy CLI + dev shell
│       ├── files.nix        # Registers nixidy manifests as sini/files entries
│       ├── inputs.nix       # flake-file input declarations
│       └── nixidy.nix       # nixidy environment definitions per cluster
├── apps/                   # nixidy application modules (cluster manifests)
│   ├── default.nix         # Shared options + imports
│   └── argocd/             # ArgoCD application
├── clusters/               # Per-cluster nixidy configuration
│   └── prod.nix
└── manifests/               # Generated output (committed for Argo CD)
    └── prod/
```

## Commands

| Command | Description |
|---|---|
| `nix run .#write-files` | Write manifests into the repo |
| `nix run .#diff-files` | Preview what would change (no writes) |
| `nix flake check` | Verify manifests + flake.nix are in sync |
| `nix run .#write-flake` | Regenerate flake.nix from module options |
| `nixidy build .#prod` | Build manifests to `./result` |
| `nixidy info .#prod` | Show environment info |

## Adding a new cluster

1. Create `clusters/<name>.nix` — set `nixidy.target.*`, `cluster.domain`, etc.
2. Add the name to `clusterNames` in `modules/flake/nixidy.nix`.
3. Run `nix run .#write-files` and commit.

## Adding a new application

1. Create `apps/<app>/default.nix` — define `applications.<app>` and `options.services.<app>`.
2. Import it in `apps/default.nix`.
3. Run `nix run .#write-files` and commit.

## Design

- **Den + flake-parts + import-tree**: modules under `modules/` are auto-discovered — no manual imports.
- **flake-file (dendritic)**: `flake.nix` is auto-generated from module options. Edit `flake-file.nix` or `modules/flake/inputs.nix`, then `nix run .#write-flake`.
- **sini/files**: manifests are declared as file entries. `nix run .#write-files` updates them; `nix flake check` verifies sync.
- **One nixidy env per cluster**: `clusters/<name>.nix` configures target repo/branch/path and cluster-specific options.
- **Application modules read `config.cluster.*`**: no hardcoding domains, CIDRs, or storage backends.
- **Helm via nixhelm**: all charts pinned in flake lock for reproducibility.
- **Host definitions and cluster creation**: future step.
